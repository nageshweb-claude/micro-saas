You are a senior full-stack engineer, product designer, and system architect.

Build a production-ready web application similar to Amazon Kindle, where users can access and read PDF books through a subscription model.

## 🎯 PRODUCT GOAL

Create a modern, fast, mobile-friendly web app where:

* Users login via OTP (no password)
* Users subscribe to access content
* Users can read PDFs in a smooth Kindle-like reader

---

## 🔐 AUTHENTICATION SYSTEM (IMPORTANT)

* ONLY OTP-based login (no passwords at all)
* Login methods:

  * Mobile number (primary)
  * Email (optional)

### Flow:

1. User enters mobile/email
2. OTP is sent (use mock or service like Firebase/Twilio)
3. User enters OTP
4. JWT/session created
5. User logged in

### Requirements:

* Secure OTP verification
* Rate limiting
* Token-based authentication (JWT)
* Auto-login persistence

---

## 💳 SUBSCRIPTION SYSTEM

* Free tier (limited access)
* Paid subscription (monthly/yearly)

### Features:

* Payment integration (Stripe / Razorpay)
* Subscription validation middleware
* Content gating (only subscribed users access full PDFs)

---

## 📚 CONTENT SYSTEM (PDF LIBRARY)

* Admin can upload PDFs
* Metadata:

  * Title
  * Author
  * Category
  * Thumbnail
  * Description

### Storage:

* Use AWS S3 / Firebase Storage

---

## 📖 PDF READER (VERY IMPORTANT - KINDLE STYLE)

Build a smooth reader experience:

### Features:

* Lazy loading pages
* Smooth scrolling / page flip mode
* Zoom in/out
* Dark mode / Light mode
* Resume from last page
* Bookmark pages
* Highlight text (optional advanced)
* Table of contents (if possible)

Use:

* PDF.js or React PDF Viewer
